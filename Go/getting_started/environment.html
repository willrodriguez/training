Environment

  1. Go to golang and download the file for your OS
  2. Close and reopen your terminal. Use go env to confirm GO is installed properly.

In 1.11 version, Go introduces the build-in package management called Go Modules which is beginning for major change Go ecosystem. It is an alternative to GOPATH with integrated support for versioning and package distribution.

The following steps are now OPTIONAL since you can now create your projects outside of $GOPATH/src

Since 1.12 version Go modules is enabled by default and the GOPATH will be deprecated in 1.13 version.

  1. Create a workspace for your Go work e.g. ~/code/github.com/nyphalanx/training/go
  2. Create a bin, pkg, and src subdirectory.
  3. Open up your bash profile subl ~/.bash_profile
  4. Add GOPATH environment variable that points to the workspace from step 1
  5. Add GOBIN environment variable that points to the workspace from step 1
  6. Update PATH environment variable to point to the "bin" subdirectory inside your go workspace 

export GOPATH=~/code/github.com/nyphalanx/training/go
export PATH="$GEM_HOME/bin:~/code/bin:$GOPATH:$PATH"
export GOBIN=~/code/github.com/nyphalanx/training/go
