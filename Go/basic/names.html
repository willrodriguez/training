Names

Names are as important in Go as in any other language. They even have semantic effect: the visibility of a name outside a package is determined by whether its first character is upper case. It's therefore worth spending a little time talking about naming conventions in Go programs.

In Go, a name is exported if it begins with a capital letter. For example, Pizza is an exported name, as is Pi, which is exported from the math package. pizza and pi do not start with a capital letter, so they are not exported.

When importing a package, you can refer only to its exported names. Any "unexported" names are not accessible from outside the package.

  • DO use mixed case (e.g. maxValue or MaxValue)
  • DO use short names (1 or 2 characters) for local variable names (e.g. mv := 5)
  • DO use long names for global variables (e.g. var maxValue int = 5)
  • DO NOT use underscores

When a package is imported, the package name becomes an accessor for the contents.

The package name is only the default name for imports; it need not be unique across all source code, and in the rare case of a collision the importing package can choose a different name to use locally.

  • DO use short names e.g. "fmt", "errors"
  • DO use a single word
  • DO use the base name of the source directory. The package in src/encoding/base64 is imported as "encoding/base64"
  • DO NOT use mix case
  • DO NOT use plurals
